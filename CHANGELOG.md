# История 

## [v2.5.2] - 2026-07-03 (Archived)
### Architecture & Code Quality
- Смешанная архитектура: сочетание MVP-подхода и монолитной структуры `core/database.py`.
- Наличие избыточного кода и временных решений («мусора»), оставшихся в процессе активной разработки (vibe-coding).

### Features & Logic
- **Кредитные карты и займы:** Бизнес-логика обработки не реализована. Функционал перенесен на версию FinanceBudgetV3 см. https://github.com/glybina18diver-lang/finance_budget_v3.
- **Сверка балансов:** Модуль не работает согласно исходному замыслу. Вероятно, будет полностью удален или заменен в будущих версиях.

### Deployment
- **Бинарный релиз:** Приложение успешно собрано в `.exe` через PyInstaller. Запуск из бинарника работает стабильно без дополнительных настроек.
